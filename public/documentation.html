API Documentation - Point of Contact for Collaboration Framework

Overview of the API designed to query a point of contact for a given product or repository within SC 1701-D.

Introduction

Software Company 1701 Delta (SC 1701-D) has seen significant growth, making collaboration more challenging. Employees, like Veena, often find it difficult to identify the right person of contact for different products, especially when teams are located in different regions.

The Point of Contact API is designed to solve this problem, allowing employees to easily find a point of contact by querying a product name or repository name.

Example Use Case

Veena is working on a feature that identifies security vulnerabilities. The team responsible for the security scan API is located in a different office, and Veena is struggling to find the right person to collaborate with. Using the Point of Contact API, Veena can easily find a person to contact by searching for the product name or repository name.

API Design and Data Model

The API was designed to query the Point of Contact for a given product or repository. The following tables were used in the database:

  • Products: Stores product information such as product name, repository, and related contact details.
  • Contacts: Contains information about contacts such as first name, last name, email, chat username, location, and title/role.

API Endpoints

GET /api/contacts

Query the Point of Contact based on either a product name or a repository name.

Query Parameters:

  • product (optional): The name of the product.
  • repository (optional): The name of the repository.

Response:

            {
                "poc_id": "uuid",
                "poc_product": "Product A",
                "poc_repository": "repo-a",
                "poc_firstName": "John",
                "poc_lastName": "Doe",
                "poc_email": "john.d@example.com",
                "poc_username": "john_d",
                "poc_location": "City, State",
                "poc_role": "Role"
            }

Error Handling:

If no matching contact is found or if an invalid query parameter is provided, the API will return an error message such as: 

            {
                "error": "No contact found for the given product or repository."
            }
